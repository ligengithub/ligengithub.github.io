    先记录几个知识点。

  • equles 和 ==
    判断两个对象相等无非就这两个操作，那么有什么区别呢？
    == 基本数据类型直接比较值，引用数据类型判断引用的地址，即对象的引用内存地址
    equles是Object的方法。默认equles也是用== 来判断的。。。那么equles意义何在，equles本没有意义，你赋予它灵魂之后，它才有意义，你不赋予它灵魂，它就等价于==

举个例子。。
两个华为mate20， 先用== 判断一下，，emmm 占用不同的空间（内存），所以是false，但是让你选一个，你使用技能equles（对比 外壳，颜色，大小）完全一模一样，equles 结果是一致为true，所以你随便选了一个。。
你看 equles 和 false 结果不一致了吧，，因为你默认的equles技能和== 技能是一样的（判断手机的位置），但是你升级了equles技能（判断手机的外观）。

总结： equles 是用来重写的，如果不重写和== 没有区别，都是判断在内存中的地址，重写之后equles就按照你的重写规则来，== 还是判断内存地址。

hashcode：也是用来重写的。。理解成身份证号
Object 有一个默认的hashCode方法，相当于默认的一个方法，可以给每个继承自Object的生成一个Hashcode代码。java中默认的hashCode返回当前对象在内存中的地址，是一个int型的数据，最大只能到2147483647，所以hashCode是可能会重复的。。

集合分类(3种)

set

无序 不重复

  • hashSet原理
    hashSet基于HashMap，hashSet的构造方法就是创建了一个hashMap，，，hashSet的元素 其实全部都存放在了hasMap的key上，，，value都是一个统一的值PRESENT是一个Object对象。添加元素的时候调用hashMap的put方法，因为hashMap添加元素会先判断key是否存在，不存在则添加，存在则添加失败。以此来保证Set的不重复性。其中的大多数方法也是采用hashMap的方法。

  • treeSet
    treeSet 是基于二叉树的Set结构，除了具备hashSet唯一性的特点，还具备有序性。
    使用treeSet添加元素元素必须具备可比较性。我们必须实现Comparator接口或者Compareable接口。
    通过重写CompareTo（Compareable）或者Compare（Object o1，Object o2）来实现排序算法
    Tree插入数据的详细过程。实现了Compareable接口以后。假设当前数集中有元素a ，我们添加元素add（b），的时候，内部会调用b.compareTo(a) 方法，所以重写的compareTo方法中this指向的是当前传入元素。如果b>a 则返回正数，b 存入a的右边。第三次add(c) 先判断c和a的大小，大于a继续判断c和b的大小，大于放右边,小于放左边。 如果c小于a，因为a的左边没有元素，所以直接放在a的左边。

注：至于是放左边还是右边，是通过比较的结果来的（返回结果为正放右边，返回结果为负放左边），即升序还是降序。

compareable和Comaperator 前者表示，对象是可以排序的，且参与排序的是同一个类的对象。就像不同人的年纪是可以比较的。可以使用List.sort()方法排序

Comperator 是一个比较器接口，重写的compare接口有两个参数，可以填写不同的对象。就像人和狗不是同一个类，但是我们可以单独比较年龄。人和狗是可以比较的。实现Comperator接口相当于创建了一个裁判，来进行比较。。而上面的comperable相当于给人类添加了一个age属性，是可以用来比较大小的。

list
  • LinkedList
    内部通过链表实现，与arrayList相反，查询遍历慢，增删快。提供操作头尾的方法，当作队列，堆栈，双向队列使用
  • ArrayList
  • 内部通过数组实现，查询和遍历快，，，增删慢。 有序
  • Vector 与ArrayList相似，数组实现，线程安全，比ArrayList慢。
map

HashMap 无序的，，默认排序是根据key的hashCode
HashTable 线程安全，效率低，继承自Dicetory类。已经被遗弃
concurrentHashMap // 锁分段技术线程安全
