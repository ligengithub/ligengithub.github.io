JAVA的NIO
Created 2019-09-09| Updated 2019-09-26

NIO 包三要素

  • Channel
  • Buffer
  • Selector
    特点: 数据总是从通道读取到缓冲区中，或者从缓冲区中写入通道。 Selector监听多个通道的事件

Channel 通道

类似于Stream，不过Stream是单向的， Channel是双向的。通道中的数据可以异步的读写，总是从一个通道读入一个Buffer，再从Buffer写入另一个通道

Channel的主要实现类有

  • FileChannel 文件
  • dataGramChannel 对应 UDP
  • SocketChannel 对应 tcp client
  • ServerSocketChannel 对应 tcp -server

    Buffer

    缓冲区，相当于连续数组
    主要有以下几种 ByteBuffer，IntBuffer，CharBuffer，LongBuffer，DoubleBuffer，FloatBuffer，ShortBuffer

Selector 选择器

核心类，监听各个通道的事件，对不同的事件做相应处理，最大的特点是管理多个连接，且在事件发生的时候才真正的去处理，减小系统开销

Buffer的几个概念

  • captain buffer总容量
  • position 当前位置
  • limit buffer可用容量

scatter 和 gather

  • scatter 将Channel的数据分散到多个buffer
  • gather 将多个Buffer的数据聚合写到一个Channel
    常用于将消息需要分散处理的场合，如消息头，消息体。

使用方法
1 将多个Buffer插入到数组，2 将数组作为参数传递给read()和write();

Selector 选择器

Selector 是NIO的一大特点,通过将多个通道绑定到一个监听器上,可以用一个线程监听多个通道。。 和Selector配合使用的通道必须是非阻塞的，
不然的话，选择器可能会阻塞在Channel的线程上，导致不能处理其它线程的响应。

几种Channel

  • ServerSocketChannel 相当于TCP服务端
  • SocketChannel 相当于TCP客户端
  • DatagramChannel udp 的Channel

注意 udp是一个无状态的协议，不能写Channel和读Channel，只能收发数据包

  • FileChannel 文件的Channel
    注意两个通道中只要有一个是FileChannel，那么就可以使用FileChannel的TransTo和TransForm方法，让数据在Channel中相互传输。
    注意，FileChannel不能配置为非阻塞

Pipe 不同线程之间通过Channel数据传输。

pipe是单向的，内部有两个Channel，分别是SourceChannel和SinkChannel，使用的时候只能把数据写入到SourceChannel中，然后从SinkChannel中读取，
两个线程之间传输数据的时候，一个线程往SourceChannel中写，另一个线程从SinkChannel中读取。

总结：

NIO和IO的比较

NIO: 非阻塞，面向缓存区,适用于多连接，小数据量。类似于聊天服务器这种。
IO: 阻塞,面向流。 适用于长连接，大数据量。大文件读写，文件下载上传。
