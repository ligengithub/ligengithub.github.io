Vert.x 入门
Created 2019-09-06| Updated 2019-09-26

Vert.x 的几个概念

  • vertx对象
    vert.x的绝大部分操作都是基于该对象。初始化参数可以用VertileOption设置

  • 调用线程 EventLoop 相当于分发任务的机器，不能阻塞，高性能的保证

  • Work verticle 工作线程。多个Work verticle 是异步的。

  • work pool Work verticle的任务会从work pool中拿线程执行

vertx的调用线程（EventLoop）
  • 默认的没有特殊说明，vertx的所有api都是非阻塞的。。如果可以返回结果就立即返回结果，如果是不能立即返回结果的，你需要定义一个hander来接收异步的返回结果。。仅仅少数以Sync结尾的，文件系统相关的api是同步的。 这个调用线程也叫做EventLoap。

  • EventLoap将请求通过EventBus 转发给处理器Hander。。

  • 一个Vertx 实例 vertx维护多个Eventloop。。通常我们根据机器的核的数量来确定EventLoop的数量

  • 注意： 即使一个Vertix实例维护多个EventLoop线程。。但是多个hander并不会被并发的执行，总是在同一个EventLoop中执行（Worker Verticle 除外）

黄金法则：不要阻塞EventLoop

阻塞的操作通常包括

  • Thread.sleep
  • 等待获取锁
  • 等待一个互斥信号或者监视器
  • 执行长时间的数据库的操作并等待其结果
  • 执行复杂计算
  • 在循环语句中长时间逗留

Verticle
部署Verticle

Verticle 差不多相当于组件。一个Work Hander或者EventLoop中为可以维护多个Verticle。 verticle必须实现Verticle接口，或者继承AbstructVerticle抽象类。

接口中有start方法和stop方法，分别在Verticle 在被装载和卸载的时候调用。

  • standard Verticle 线程安全的，同步的，会分配给EnventLoop线程，并由同一个EventLoop线程执行该Hander。

  • Worker Verticle 线程安全，异步的。 将会分配给Worker Pool线程池中的线程调用，同一时刻只有一个线程调用。也是线程安全的，通过设置Worker(true)

option 可以设置instance数量，在多核cpu上很有用。直接设置多个instance，来充分利用多核。

  • multi-threaded Worker Verticle 线程不安全，需要用户自己保证。
