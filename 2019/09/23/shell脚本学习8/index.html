shell 重定向(八)
Created 2019-09-23| Updated 2019-09-26

输入输出重定向

每一个Shell命令的执行都包含3个步骤
1 从stdin（标准输入读取） 文件描述符 0
2 输出到stdout（标准输出）文件描述符为 1
3 如果发生异常输出到stderr 文件描述符为 2

命令分类

  • 2 command >file 输出重定向到文件
    最简单的

会生成一个testfile的文件，文件的内容是xixixixixi,如果没有>testfile 那么将会在控制台打印xixixixixi,>将输出从stdout重定向到了file，所以这个命令的结果就输出到了file里面
本质上是从stdout重定向到testfile

  • 2 command <file 输入重定向到文件
    文件重定向到输入

wc 指令 word count 统计文件中的字节数(-c)，行数(-l)，字数(-w),字符数(-m),最长行长度(-L)

  • command >> file 以追加的方式将输出重定向到文件
    这个很好理解 使用>的时候，每次执行会覆盖掉前一次的。。使用>> 就是在原来的基础上追加。
  • n>file 将文件描述符为n的文件重定向到file
    文件描述符标识一个文件，入stdout 标准输出的文件标识符是1

  • n>>file 将文件描述符为n的文件以追加的方式重定向到file
    追加重定向输出到file

  • n>& m 将输出文件m和n合并

  • n<&m 将输入文件m和n合并

  • <<tag 将开始标记tag和结束标记tag之间的内容做为输入

/dev/null 文件

希望执行某个命令，但是又不希望再屏幕上显示输出，可以将输出重定向到该目录下。
这个是一个特殊的文件，写到它的内容都会被丢弃，如果尝试从该文件读取内容，什么也读取不到。
