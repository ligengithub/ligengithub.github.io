redis 没有开启持久化情况下的迁移数据库

这种情况下我们就需要手动进行持久化了，然后重复上述1 步骤了。

手动持久化的方式,进入容器，启动redis-cli连接数据库，执行BGREWRITEAOF 命令。这条指令的意思是重写aof持久化文件，
，就是将当前redis中的数据以最少的aof脚本备份。 可以在你即使启动的时候没有开启持久化，
也可以在你的挂载目录下生成一个持久化的文件。

然后重复上述1 步骤即可

但是注意，着个只是根据当前的数据生成了一个aof持久化文件，并不会开启持久化。
你重启docker 虽然aof还在，但是并不能获取到持久化的数据，
