JAVA的IO
Created 2019-09-16| Updated 2019-09-26

IO 操作是面向流的，首先流有两种 字节流和字符流，字节流和字符流又各有两种， 输入流和输出流。

常用的类

如下图
常用的只有5个类和一个接口

  • InputStream 字节输入流
    （二进制格式操作）抽象类，基于字节的输入操作，所有输入流的父类。
  • OutputStream 字节输出流
    （二进制格式操作）抽象类，基于字节的输出操作，所有输入流的父类。
  • Reader 字符输入流
    (文件格式操作) 基于字符
  • Writer 字符输出流
    (文件格式操作) 基于字符
  • File 文件类

文件类，文件，目录等信息，色恒诚新目录，文件的增删改查。

  • RandomAccessFile 随机存取文件类
    文件类的一个工具类，继承自File类
    常用的就这6个类，以及Serializable接口。基本上可以完成大部分的io操作了。那剩下的是什么作用呢？
    我们呢分层来看

类的分类，结构

  • 流式部分，以Stream 结尾的类。是IO操作的核心。
  • 非流式部分 像RandomAccessFile，FileDescriptor 等是辅助类。
  • 其它有文件读写，安全相关的类。SerializablePermission类，FileSystem类等。

结构如下图

流的分类

  • File（文件）： FileInputStream,FileOutputStream,FileReader,FileWriter
  • byte[] : ByteArrayInputStream,ByteArrayOutPutStream
  • Char[] : CharArrayReader,CharArrayWriter
  • String: StringBufferInputStream,StringReader,StringWriter
  • 网络数据流 InputStream,OutputStream,Reader,Writer

字节流和字符流

  • 字节流 ; 未经处理的最原始数据
  • 字符流 : 最小单元式字符，java中字符的编码式Unicode码，一个字符两个字节。

为什么要有字符流，因为java中字符式Unicode编码，一个字符16为，为了高效的处理字符，引入了字符流。
本质还是将字节流读入读出的时候去查了指定的码表。

输入流和输出流

  • 输入流 ： 将数据从存储介质或者外部设备读取到程序中
  • 输出流 ：将数据写入存储介质或者将数据输出到外部设备。

字节流和字符流的转换

  • 连接字节流和字符流的桥梁
  • 将字节编码成字符
  • 字符解码成字节

转换流

OutputStreamWriter 字节流以字符流输出
InputStreamReader 字节流以字符流输入

这两个流，有转换作用，本身又是字符流，在钩爪参数需要传入字节流对象。

字节流和恶字符流的区别，以及选择

  • 字节流没有缓冲区，直接输出的。字符流是到缓冲区，调用close的时候才会输出，或者调用flush（）
  • 读写单位不同，字节流8bit为处理单位，字符流以字符为单位处理，根据码表进行映射，一次可能读取多个字节
  • 处理对象，字符流可以处理任何类型的数据，图片，视频。。。 字符流只能处理字符类型的数据。

总结： 除了处理纯文本的文件用字符流，其它类型呢的统一使用字节流。
