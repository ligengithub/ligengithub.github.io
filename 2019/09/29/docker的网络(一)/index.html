docker的网络

docker进程启动以后，在我们的主机上会出现一个docker0的虚拟网卡，ip是127.0.0.1，子网掩码是255.255.0.0
意味着docker 可以创建ip从172.17.0.2-172.17.255.255 的网段，共65534个ip。 172.17.0.1 docker网桥网关

所以桥接的意思就是，容器使用docker0下面的网段(172.17.0.2-172.17.255.255)。桥接模式生成的容器，ip就在这个网段下面了。

3 host网络

host网络就是将容器直接使用宿主机的网络，也就是说容器实际和宿主机是同一个ip。 比如我的宿主机的ip是192.168.1.1 ，容器的ip也就是192.168.1，
端口实际上就是占用宿主据的监听端口，以host方式启动的容器只能监听不同的端口。比如桥接模式的时候，我们可以指定容器1 -p 8080:8080，容器2 8081：8080
把宿主机的不同端口映射到容器的相同端口。但是host方式启动是不行的。容器直接占用宿主机的端口。

4 null 网络

null网络就是创建一个什么都没有的网络。这个容器不能进行网络通信，也不能和其它容器使用link连接通信（什么是link，看下回分解）。
就相当于是一个与世隔绝的容器。那么它有什么用。
网上说，用于部署一些安全性非常高的应用，大概是核弹发射密码这种的把。
