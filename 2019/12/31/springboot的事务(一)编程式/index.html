Springboot的事务(一)编程式
Created 2019-12-31| Updated 2020-01-15

事务的分类

1 编程式事务
 什么是编程式事务，我们通过编程的方法，
手动的去创建一个事务。手动的去提交和回滚。这个就叫编程式事务，
通常在我们使用 TransactionTemplate和PlatformTransactionManager这两个类可以去创建事务,
spring推荐我们使用TransactionTemplate。从名字就可以看出来，事务模板。

2 声明式事务
为什么叫声明式事务，通俗的来说，就是使用注解@Transaction去声明，
     该注解下的方法使用事务。当你某个方法，需要事务时候，，你只需要在上面添加一个@Transaction注解声明一下即可。

 3 两者比较和区别
 编程式事务
 优点: 事务的作用区域我们可以自己控制，加载代码中的某一个代码段。
 缺点，不方便，和我们的代码的耦合性高。
 声明式事务
 优点： 简单，不会侵入代码，只需要加注解即可
 缺点： 最细粒度只能作用到方法级别，不能自定义回滚执行一些自己的操作。

编程式事务的两种方式

1 PlatformTransactionManager
它是一个比较底层的事务管理器接口，我们通常不直接使用它，而是使用它的实现类DataSourceTransactionManager。我们可以看到他们关系如下
![](https://i.loli.net/2020/01/02/HNECsae64dRSljz.png)
PlatformTransactionManager接口只有三个核心方法
-  TransactionStatus getTransaction( TransactionDefinition definition)  // 获取事务
-  void commit(TransactionStatus status)    // 提交
-  void rollback(TransactionStatus status)  // 回滚

TransactionTemplate
我们也可以使用TransactionTemplate，它的使用方法如下


execute方法参数我们传入一个TransactionCallback函数是接口，接口方法是doInTransaction()，从名字也可也可以看出来，在事务中执行。同时，改接口还有一个抽象子类TransactionCallbackWithoutResult
没有返回结果，当我们不需要返回值的时候可以使用该方法，使用如下
/**
 * @ desc :  方式2  TransactionTemplate
 * @ params
 * @ return   无返回值
 * @ date 2020/1/1
 * @ author ligen
 */
public void test3() {
    transactionTemplate.execute(new TransactionCallbackWithoutResult() {
        @Override
        protected void doInTransactionWithoutResult(TransactionStatus status) {
            try {
                jdbcTemplate.execute("insert into info values(null,\"ligen2\")");
                throw new Exception();
            } catch (Exception e) {
                System.out.println("TransactionTemplate 无返回值异常回滚"); 
                status.setRollbackOnly();
            }
        }
    });
}

总结

以上就是springboot的编程式事务的两种简单使用方式，DataSourcesTransactionManager 和 TransactionTemplate的还有一个区别就是，前者可以将事务的执行结果同步的返回回去，而后者不能

注意不能在@Transaction的注解作用范围内使用编程式事务，不然会报错。
